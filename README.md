# MiniCompiler — Учебный компилятор для MiniLang

##  О проекте

**MiniCompiler** — это учебный компилятор для упрощенного C-подобного языка **MiniLang**, реализованный на C++17. Проект создан для изучения принципов построения компиляторов и включает полный цикл обработки исходного кода: от лексического анализа до построения абстрактного синтаксического дерева (AST).

###  Цели проекта
- **Образовательная:** изучение теории компиляции на практике
- **Модульная:** четкое разделение этапов компиляции
- **Тестируемая:** полное покрытие тестами
- **Расширяемая:** возможность добавления новых возможностей

###  Ключевые особенности

#### Спринт 1 (Лексический анализ) 
- Полноценный лексический анализатор с поддержкой всех типов токенов
- Препроцессор с поддержкой макросов и условной компиляции (`#define`, `#ifdef`, `#if`, `#elif`, `#else`, `#endif`)
- Подробная обработка ошибок с указанием позиции и кода ошибки
- 36 автоматических тестов (25 валидных + 11 с ошибками)
- Производительность O(n), поддержка файлов до 1 МБ
- Поддержка escape-последовательностей в строках (`\n`, `\t`, `\r`, `\"`, `\\`)
- Вложенные комментарии `/* /* */ */`
- Восстановление после ошибок (panic mode)

#### Спринт 2 (Синтаксический анализ) 
- Парсер рекурсивного спуска с поддержкой LL(1) грамматики
- Полная иерархия узлов AST (Abstract Syntax Tree)
- Поддержка всех операторов с правильным приоритетом и ассоциативностью
- Объявления функций, структур и переменных
- Управляющие конструкции: `if-else`, `while`, `for`, `return`
- Вызовы функций с аргументами
- Индексация массивов `arr[10]`
- Доступ к полям структур `p.x`
- Постфиксные операторы `x++`, `x--`
- Составные присваивания `+=`, `-=`, `*=`, `/=`, `%=`
- Визуализация AST в текстовом формате и Graphviz DOT
- Интеграция с лексером и препроцессором
- 9 тестов парсера + 5 тестов синтаксических ошибок

##  Быстрый старт

### Установка и сборка
```bash
# Клонирование репозитория
cd minicompiler

# Сборка проекта
make clean
make

# Проверка работы
./minicompiler help
```

### Первая программа
Создайте файл `hello.mini`:
```c
// Моя первая программа на MiniLang
fn main() -> int {
    int x = 42;
    string msg = "Hello, World!";
    return x;
}
```

Запустите компиляцию:
```bash
# Только лексер (показать токены)
./minicompiler lex hello.mini

# Парсер с выводом AST
./minicompiler parse hello.mini

# Сгенерировать DOT для визуализации
./minicompiler parse hello.mini --format dot --output ast.dot

# Визуализировать (требуется graphviz)
dot -Tpng ast.dot -o ast.png
eog ast.png
```

##  Команды и использование

### Основные команды
```bash
# Сборка проекта
make all

# Очистка
make clean

# Токенизация файла (только лексер)
./minicompiler lex <file>

# Только препроцессор
./minicompiler preprocess <file>

# Парсинг и построение AST
./minicompiler parse <file> [options]

# Препроцессор + лексер + парсер
./minicompiler compile <file>

# Справка
./minicompiler help
```

### Опции для команды parse
```bash
--format <text|dot|json>   # Формат вывода AST (по умолчанию: text)
--output <file>            # Выходной файл (по умолчанию: stdout)
--verbose                  # Показать токены и отладочную информацию
--stats                    # Показать статистику компиляции
```

### Makefile цели
| Команда | Описание |
|---------|----------|
| `make all` | Собрать все компоненты |
| `make clean` | Очистить объектные файлы |
| `make test-lexer` | Запустить тесты лексера (Спринт 1) |
| `make test-parser` | Запустить тесты парсера (Спринт 2) |
| `make test-integration` | Тестирование интеграции на примерах |
| `make test-all` | Запустить все тесты |
| `make test-preprocess` | Тестирование препроцессора |
| `make ast-factorial` | Визуализация AST для factorial.src |
| `make ast-file FILE=x` | Визуализация для указанного файла |
| `make ast-all` | Все визуализации |
| `make check` | Полная проверка проекта |
| `make help` | Показать справку |

##  Примеры использования

### Пример 1: Функции и арифметика
```c
fn add(int a, int b) -> int {
    return a + b;
}

fn main() -> int {
    int x = add(5, 3);
    int y = x * 2;
    return y;
}
```

### Пример 2: Структуры и доступ к полям
```c
struct Point {
    int x;
    int y;
}

fn main() -> int {
    Point p;
    p.x = 10;
    p.y = 20;
    return p.x + p.y;
}
```

### Пример 3: Управляющие конструкции
```c
fn factorial(int n) -> int {
    int result = 1;
    while (n > 0) {
        result = result * n;
        n = n - 1;
    }
    return result;
}
```
