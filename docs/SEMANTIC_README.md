# Документация семантического анализатора

## О проекте

Семантический анализатор - третий компонент компилятора MiniLang, отвечающий за проверку смысловой корректности программы. Он анализирует абстрактное синтаксическое дерево (AST), построенное парсером, и проверяет:
- Объявления переменных и функций
- Области видимости
- Соответствие типов
- Корректность вызовов функций
- Правильность использования операторов
- **Memory Layout** — вычисление смещений переменных в стеке

---

## Быстрый старт

### Сборка проекта
```bash
# Очистка и сборка
make clean
make

# Проверка, что семантический анализатор работает
./minicompiler check examples/factorial.src --verbose
```

### Базовое использование
```bash
# Семантический анализ с выводом таблицы символов
./minicompiler check <filename> --verbose

# Семантический анализ с выводом статистики
./minicompiler check <filename> --stats

# Полная компиляция (препроцессор + лексер + парсер + семантика)
./minicompiler compile <filename>
```

---

## Команды программы

| Команда | Описание | Пример |
|---------|----------|---------|
| `check <file>` | Семантический анализ | `./minicompiler check factorial.src` |
| `check <file> --verbose` | Подробный вывод (токены + AST + символы + Memory Layout) | `./minicompiler check factorial.src --verbose` |
| `check <file> --stats` | Показать статистику | `./minicompiler check factorial.src --stats` |
| `compile <file>` | Полная компиляция | `./minicompiler compile program.src` |

### Иерархия таблицы символов

```
SymbolTable
├── Глобальная область видимости (уровень 0)
│   ├── Функции
│   └── Структуры
├── Область видимости функции (уровень 1)
│   ├── Параметры
│   └── Локальные переменные
└── Вложенные блоки (уровни 2+)

```

| `bool` | 1 | 1 |
| `string` | 8 | 8 |
| `void` | 0 | 0 |
| `struct` | сумма полей | максимальное выравнивание полей |

## Проверки семантического анализатора

### 1. Проверка объявлений

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Дублирование | Нельзя объявить дважды в одной области | `int x; int x;` |
| Типы | Проверка корректности типов | `void x = 5;` |
| Структуры | Проверка полей структуры | `struct Point { int x; int x; }` |

### 2. Проверка областей видимости

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Поиск символов | Ищет от внутренней к внешней области | `x = 5;` (x не объявлен) |
| Вложенные области | Переменные во вложенных блоках | `{ int x; } x = 5;` |

### 3. Проверка типов

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Присваивание | LHS тип должен быть совместим с RHS | `int x = "hello";` |
| Бинарные операторы | Проверка типов операндов | `int x = true + 5;` |
| Унарные операторы | Проверка типа операнда | `bool x = !5;` |
| Расширение типов | int → float разрешено | `float x = 5;` |

### 4. Проверка функций

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Количество аргументов | Соответствие объявлению | `add(5);` (ожидается 2) |
| Типы аргументов | Совместимость с параметрами | `add(5, "hello");` |
| Возвращаемый тип | Соответствие объявлению | `int fn() { return "x"; }` |
| Отсутствие return | Функция с типом должна возвращать значение | `int fn() { }` |

### 5. Проверка управляющих конструкций

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Условие if | Должно быть типа bool | `if (5) { }` |
| Условие while | Должно быть типа bool | `while (x) { }` |
| Условие for | Должно быть типа bool | `for (; x; ) { }` |

### 6. Проверка массивов

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Индексация | Только для массивов | `int x = 5; x[0] = 1;` |
| Тип индекса | Должен быть int | `arr["hello"] = 1;` |

### 7. Проверка структур

| Проверка | Описание | Пример ошибки |
|----------|----------|---------------|
| Доступ к полям | Поле должно существовать | `p.z = 5;` (z не существует) |
| Тип объекта | Должен быть структурой | `int x; x.y = 5;` |

## Таблица кодов ошибок

### Лексические ошибки (LEX-xxx)
| Код | Описание | Подсказка |
|-----|----------|-----------|
| LEX-001 | Недопустимый символ | Удалите или замените недопустимый символ |
| LEX-002 | Неверное целое число | Используйте только цифры для целых чисел |
| LEX-004 | Незакрытая строка | Добавьте закрывающую кавычку " в конце строки |
| LEX-005 | Новая строка в строке | Используйте \n для переноса строки |
| LEX-006 | Неверная escape-последовательность | Допустимые: \n \t \r \" \\ |
| LEX-008 | Слишком длинный идентификатор | Сократите имя до 255 символов |

### Синтаксические ошибки (SYN-xxx)
| Код | Описание | Подсказка |
|-----|----------|-----------|
| SYN-001 | Пропущена точка с запятой | Добавьте ';' в конце оператора |
| SYN-002 | Пропущена скобка | Добавьте закрывающую скобку ')' |
| SYN-003 | Пропущена фигурная скобка | Добавьте закрывающую скобку '}' |
| SYN-004 | Неожиданный токен | Проверьте синтаксис вокруг этого токена |
| SYN-005 | Ожидался идентификатор | Укажите имя переменной или функции |
| SYN-006 | Ожидался тип | Укажите тип (int, float, bool, string) |

### Семантические ошибки (SEM-xxx)
| Код | Описание | Подсказка |
|-----|----------|-----------|
| SEM-001 | Необъявленный идентификатор | Объявите переменную перед использованием |
| SEM-002 | Повторное объявление | Используйте другое имя или удалите предыдущее |
| SEM-003 | Несоответствие типов | Проверьте соответствие типов в выражении |
| SEM-004 | Необъявленная функция | Объявите функцию перед вызовом |
| SEM-005 | Неверное количество аргументов | Проверьте количество аргументов при вызове |
| SEM-006 | Неверный тип аргумента | Проверьте типы аргументов |
| SEM-007 | Несоответствие типа возврата | Функция должна возвращать указанный тип |
| SEM-008 | Отсутствует return | Добавьте оператор return в функцию |
| SEM-009 | Неверный тип условия | Условие должно быть типа bool |
| SEM-010 | Неверный индекс массива | Индекс должен быть целым числом |
| SEM-011 | Индексация не-массива | Оператор [] только для массивов |
| SEM-012 | Поле не найдено | Поле не существует в структуре |

```bash
fn main() -> int {

```
